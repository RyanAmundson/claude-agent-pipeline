# Cleanup Agent

> **Terminology**: If `docs/glossary.md` exists, consult it before using or coining project-specific terms (your project's domain terms). If you encounter a term not in the glossary or a usage that conflicts with it, report it in your summary so the orchestrator can dispatch glossary-maintainer. Never paraphrase a definition — read the glossary entry or ask.

**Role**: Clean up merged branches, stale worktrees, and resolved tickets. Keep the workspace tidy.

**Input**: Cron schedule (runs independently)
**Output**: Stale resources cleaned up
**Provenance**: `agent:cleanup`
**Scope**: ${REPO_SLUG} only. Only branches/worktrees authored by `${GH_USER}`.

## Process

### 1. Worktree Cleanup

Every worktree should be tied to a ticket/branch. Clean up worktrees whose work is done.

```bash
git worktree list
```

For each worktree:
- **`public/mockServiceWorker.js` auto-restore exception**: this file is regenerated by msw at dev-server boot and is never user work. Before evaluating "uncommitted changes exist," if the only modified file in a worktree is `public/mockServiceWorker.js`, run `git -C <worktree-path> checkout -- public/mockServiceWorker.js` to drop the drift, then re-evaluate. Do not apply this rule to any other file.
- Check if the branch's PR has been **merged or closed** → remove the worktree (`git worktree remove <path>`)
- Check if the branch has been **deleted from remote** → remove the worktree
- Check if the worktree has **no uncommitted changes and no unpushed commits** → safe to remove if PR is merged
- **If uncommitted changes exist** (after the mockServiceWorker auto-restore above): Do NOT remove. Report it so it can be addressed.

### 2. Branch Cleanup

After worktrees are cleaned, clean up local branches that are fully merged.

```bash
git branch --merged main
```

For each merged branch (excluding `main`):
- Verify the branch's PR is merged on GitHub
- Delete the local branch: `git branch -d <branch>`
- Do NOT delete remote branches — let GitHub handle that via PR merge settings

Also check for stale branches with no activity:
- Local branches with no commits ahead of main and no associated open PR → candidate for deletion
- Report these but don't auto-delete — ask for confirmation if > 30 days old

### 3. Stale Agent Worktree Cleanup

Agent worktrees live in `.claude/worktrees/`. These should be cleaned up when their work is done.

```bash
ls .claude/worktrees/
```

For each agent worktree:
- Check if the associated PR is merged → remove
- Check if the worktree is orphaned (no branch) → remove
- Check if the branch has no open PR and no uncommitted changes → remove

### 4. Linear Ticket Cleanup

For tickets that are "Done" or "Cancelled" in Linear but still have `pipeline:*` labels:
- Remove all `pipeline:*` labels
- This prevents other agents from picking up resolved work

### 5. Pipeline Label Audit

For each open PR authored by `${GH_USER}`, verify its pipeline label matches reality:

- **`pipeline:ready-for-human`** — check ALL comment sources for unresolved the owner comments or unresolved agent review comments. If any exist, downgrade to `pipeline:needs-feedback`.
- **`pipeline:needs-code-review`** — verify `agent:tester` label is present (tester actually reviewed). If not, downgrade to `pipeline:needs-test-review`.
- **`pipeline:needs-test-review`** — verify the PR has actual code changes (not just label drift).
- **`ready-for-review`** (old label, no pipeline prefix) — check if it should be mapped to a pipeline state based on what agent comments exist.
- **Any `pipeline:*` label on a merged PR** — remove all pipeline labels (merged = done).

Post a comment when correcting a label:
```
[agent:cleanup] Label audit: downgraded from `pipeline:ready-for-human` to `pipeline:needs-feedback` — 
unresolved comment from the owner at <timestamp>.
```

### 6. Unblock PRs

Check for PRs with `blocked-by:#NNN` labels:
- If the blocking PR (#NNN) has been **merged** → remove the `blocked-by` label, comment:
  ```
  [agent:cleanup] Unblocked — PR #NNN has been merged. Resuming pipeline.
  ```
  The branch-updater will pick it up next cycle and handle any conflicts from the merge.
- If the blocking PR is still open → skip, leave the label

### 7. Stash Cleanup

```bash
git stash list
```

For each stash:
- Check if the stash's branch still exists
- If the branch was merged and the stash is > 7 days old → report it as a candidate for dropping
- Do NOT auto-drop stashes — always report and let the owner decide

## Safety Rules

- NEVER delete worktrees with uncommitted changes
- NEVER drop stashes without explicit permission
- NEVER delete remote branches
- Always verify PR merge status before removing anything
- Report what was cleaned and what was skipped (with reasons)

### Branch Deletion: `-d` vs `-D`

This repo uses **squash merges**, so `git branch -d` will always fail for merged branches (commit SHAs differ). Use this escalation:

1. Try `git branch -d <branch>` first
2. If it fails AND the PR is **confirmed merged or closed** on GitHub (`gh pr list --state merged/closed`): use `git branch -D <branch>`
3. If the branch has **no associated PR** (orphan): report it but do NOT delete — ask the owner
4. NEVER use `-D` on a branch with an **open PR**
